# Packet Module Guidelines

`packet` モジュールは、Minecraft のパケット送受信を DSL やリスナーを介して簡単に行うためのライブラリです。

## 1. パケットリスナー (PacketListener)
- `PacketListener` を介して、パケットの送信 (`onSend`)、受信 (`onReceive`) をフックする。
- Netty レベルでの横取りと書き換え (`transformSend`, `transformReceive`) をサポートし、低レベルな制御も可能にする。
- 各パケット型に対し、インライン関数と `reified` を活用して型安全にアクセスできるように設計する。

## 2. Packet DSL
- `packet(player) { ... }` を入り口とし、タイトルの表示、音の再生、フェイクエンティティの生成などを宣言的に記述できる。
- `SoundBuilder`, `TitleBuilder`, `FakeEntityBuilder` などの Builder クラスを通じて詳細な設定を行う。
- `player.onPacket<PacketType> { ... }` などの拡張関数を提供し、パケット処理を簡素化する。

## 3. リフレクションとキャッシュ
- パケットの内部フィールド（NMS 独自のものなど）にアクセスする際は、リフレクションを使用する。
- 頻繁にアクセスするフィールドは、`PacketWrapper.kt` 内のキャッシュ機構（`fieldCache`）を介してパフォーマンスを維持する。

## 4. バージョン対応 (Multi-Version Support)
- 現在、`v1_20_4`, `v1_20_6`, `v1_21_4`, `v1_21_11` などのサブプロジェクトでバージョン差異を吸収している。
- `InternalAPI` は、共通 API からバージョン固有の実装を呼び出すためのハブとして機能する。
- パケットクラスのコンストラクタ変更などは、それぞれのサブモジュール内で解決し、ユーザーには共通のインターフェースのみを公開する。

## 5. 注意事項
- `FriendlyByteBuf` などを直接操作して生パケットを送る際は、サーバーバージョンによるバイナリ形式の違いに十分注意すること。
- フェイクエンティティなどの操作は、パケットレベルの副作用を伴うため、適切にライフサイクル管理（パケットの送信タイミング、リソースの解放など）を行うこと。
