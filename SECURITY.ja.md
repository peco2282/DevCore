# セキュリティポリシー

このセキュリティポリシーは、**DevCore** プロジェクトリポジトリ内のすべてのモジュールに適用されます。これには、以下のモジュールが含まれます（ただしこれらに限定されません）。
- `core`, `adventure`, `command`, `config`, `scheduler`, `packet`, `gui`, `event`, `cooldown`, `database`, `effect`, `entity`, `util`, `world`, および `scoreboard`（NMS サブモジュールを含む）。

## サポートされているバージョン

DevCore の現在のメジャーバージョン、およびアクティブにターゲットとしている Minecraft / Paper バージョンに対してセキュリティアップデートを提供します。

| バージョン | サポート状況         |
| ---------- | -------------------- |
| 1.x.x      | :white_check_mark:   |
| < 1.0.0    | :x:                  |

### ターゲットとする Minecraft / Paper バージョン

DevCore は現在、以下のバージョン向けの実装を提供しています。
- **1.21.x** (例: 1.21.4, 1.21.1)
- **1.20.6**

## 脆弱性の報告

本プロジェクトでセキュリティの脆弱性を発見した場合は、速やかに報告してください。

### 報告方法

1. **GitHub の非公開脆弱性報告 (Private Vulnerability Reporting):**
   GitHub の [セキュリティアドバイザリ (Security Advisory)](https://github.com/peco2282/DevCore/security/advisories/new) 機能を使用して、非公開で脆弱性を報告してください。これが推奨される方法です。

2. **代替方法:**
   GitHub の報告機能を使用できない場合は、[README.md](README.md) またはプロジェクトのウェブサイトに記載されている連絡先を通じて、プロジェクトの管理者に連絡してください。

### 報告に含める内容

問題を迅速に調査・解決するために、以下の情報を提供してください。
- 脆弱性の明確な説明。
- 問題を再現するための手順。
- 潜在的な影響と深刻度。
- 修正案や緩和策（もしあれば）。

### 対応方針

- **受付確認:** 報告を受領してから 48〜72 時間以内に確認の連絡をします。
- **調査:** 問題を調査し、その影響を判断します。
- **解決:** 脆弱性が確認された場合、タイムリーに修正を行い、リリースします。
- **公開:** 修正がリリースされた後、報告者と協力して公開情報の調整を行います。

**重要:** セキュリティの脆弱性を GitHub のパブリックな Issue や Discussion で報告しないでください。上記の非公開な報告方法を使用してください。
